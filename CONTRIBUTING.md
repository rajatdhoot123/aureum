# Contributing to Aureum (RateWatch)

Thank you for your interest in contributing to Aureum! This guide will help you get started.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/RateWatch.git
   ```
3. **Create a feature branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Setup

- **Android Studio** Hedgehog (2023.1.1) or higher
- **JDK 17** or higher
- **Android SDK 35**

See the [README.md](README.md) for full setup instructions.

## How to Contribute

### 🐛 Bug Reports
- Search existing [issues](https://github.com/aureum-app/RateWatch/issues) before opening a new one.
- Use the bug report template when available.
- Include steps to reproduce, device info, and screenshots if applicable.

### ✨ Feature Requests
- Open a discussion issue with the label `enhancement`.
- Describe the use case and why it would benefit Indian investors.
- Be open to feedback and alternative approaches.

### 🔧 Pull Requests
- Keep PRs focused — one feature or fix per PR.
- Follow existing code style and conventions.
- Write clear commit messages.
- Include tests for new functionality (if applicable).
- Update documentation as needed.

### 📖 Documentation
- Improve docstrings, comments, and guides.
- Translate README into supported Indic languages.
- Add examples or tutorials.

## Code Style

- Follow Kotlin official style guides.
- Use meaningful variable and function names.
- Keep functions small and focused.
- Add Compose previews where possible.

## Reporting Security Issues

**Please do not report security vulnerabilities in public issues.** Email maintainers directly with details.

## Questions?

Feel free to open a discussion or reach out via GitHub.
